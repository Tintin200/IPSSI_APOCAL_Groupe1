# Release Planning mis à jour

## Objectif

Le release planning est mis à jour pour intégrer les nouvelles exigences de la perturbation 5.

## Release 1 — Stabilisation du produit

Objectif : conserver les fonctionnalités principales et éviter une régression.

Contenu :

- accès à la plateforme ;
- consultation des contenus ;
- interaction avec l'IA ;
- correction des anomalies bloquantes.

## Release 2 — Accessibilité et internationalisation

Objectif : rendre la plateforme utilisable par un public plus large.

Contenu :

- audit RGAA ;
- navigation clavier ;
- amélioration des contrastes ;
- externalisation des textes ;
- ajout d'un sélecteur de langue ;
- paramètre de langue pour les réponses IA.

## Release 3 — Scalabilité et résilience

Objectif : préparer la plateforme à une adoption nationale.

Contenu :

- tests de charge ;
- cache ;
- surveillance des performances ;
- fournisseur LLM de secours ;
- réflexion sur une architecture scalable.

## Impact sur le planning

La perturbation augmente le périmètre du projet.  
Certaines fonctionnalités secondaires doivent être reportées pour prioriser :

- l'accessibilité ;
- la disponibilité du service ;
- la capacité à gérer plusieurs langues.
